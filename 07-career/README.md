# §7 — Career & Portfolio

**Goal:** Package your learning into career assets that demonstrate real AI engineering skills.

## Learning Objectives

- Transform projects into polished portfolio pieces
- Build public presence through writing and open-source contributions
- Create demonstrable evidence of AI engineering competence

---

## Action Items

This section has no code projects. Instead, it's a checklist of high-impact career moves:

### 🎨 Polish Your Portfolio (Pick 2-3 Projects)

Take your best projects from this roadmap and make them portfolio-ready:

- **Professional README** — problem statement, architecture diagram, demo GIF/video, setup instructions
- **Clean code** — remove dead code, add docstrings, organize imports
- **Architecture diagram** — even a simple Mermaid diagram shows you think about design
- **Demo** — record a 2-3 minute walkthrough (Loom, OBS, or asciinema for CLI tools)

### ✍️ Write About What You Built (2-3 Posts)

Blog posts are career multipliers. Write about:

- A technical deep-dive into one of your projects
- Lessons learned (what went wrong, what you'd do differently)
- A comparison or benchmark you ran (models, chunking strategies, etc.)

Publish on Dev.to, Hashnode, or your own blog. Cross-post to LinkedIn.

### 🤝 Contribute to Open Source (1 Project)

Pick one open-source AI project and make a meaningful contribution:

- [Langfuse](https://github.com/langfuse/langfuse) — LLM observability
- [LiteLLM](https://github.com/BerriAI/litellm) — unified LLM API proxy
- [Instructor](https://github.com/jxnl/instructor) — structured outputs
- [ChromaDB](https://github.com/chroma-core/chroma) — vector database
- [FastAPI](https://github.com/tiangolo/fastapi) — the web framework you've been using

Start with documentation fixes or small bug fixes. Then tackle a feature.

### 🏗️ Build Your GitHub Profile

- Create a GitHub profile README showcasing your roadmap completion
- Pin your best 4-6 repositories
- Make sure each pinned repo has a clear description and topic tags

### 🎥 Record a Demo Video

- Pick your best project
- Record a 5-minute walkthrough: problem → architecture → demo → lessons learned
- Upload to YouTube (unlisted is fine) and link from your portfolio

## Key Concepts

After completing this section, you should have:

- 2-3 polished portfolio projects with professional READMEs
- 2-3 published blog posts about AI engineering
- At least 1 open-source contribution
- A strong GitHub profile
- A demo video showcasing your best work
- Confidence to discuss AI engineering in interviews
